## AI（人工知能）

### AIの分類

| 種類 | 説明 |
|------|------|
| 弱いAI（特化型AI） | 特定タスクに特化 |
| 強いAI（汎用AI） | 人間と同等の知能（未実現） |

### 機械学習の種類

| 種類 | 説明 | 例 |
|------|------|-----|
| 教師あり学習 | 正解ラベル付きデータで学習 | 分類、回帰 |
| 教師なし学習 | ラベルなしデータで学習 | クラスタリング |
| 強化学習 | 報酬を最大化するよう学習 | ゲームAI、ロボット |

### 主要アルゴリズム

**教師あり学習**
| アルゴリズム | 用途 |
|-------------|------|
| 線形回帰 | 数値予測 |
| ロジスティック回帰 | 二値分類 |
| 決定木 | 分類・回帰 |
| ランダムフォレスト | アンサンブル学習 |
| SVM | 分類 |

**教師なし学習**
| アルゴリズム | 用途 |
|-------------|------|
| k-means | クラスタリング |
| 主成分分析（PCA） | 次元削減 |

### ディープラーニング

多層ニューラルネットワークによる学習。

| アーキテクチャ | 用途 |
|---------------|------|
| CNN（畳み込みNN） | 画像認識 |
| RNN/LSTM | 時系列データ、自然言語 |
| Transformer | 自然言語処理（BERT, GPT） |
| GAN | 画像生成 |

### 自然言語処理（NLP）

| 技術 | 説明 |
|------|------|

| 機械翻訳 | 言語間の翻訳 |

### 画像認識

| 技術 | 説明 |
|------|------|

## IoT（Internet of Things）

### IoTアーキテクチャ

```
┌──────────────┐
│   クラウド    │ ← 分析・可視化
├──────────────┤
│  エッジ/ゲートウェイ │ ← データ前処理
├──────────────┤
│   デバイス/センサー  │ ← データ収集
└──────────────┘
```

### 通信プロトコル

| プロトコル | 特徴 | 用途 |
|-----------|------|------|
| MQTT | 軽量、Pub/Sub | センサーデータ送信 |
| HTTP/HTTPS | 汎用的 | API通信 |
| CoAP | 軽量、UDP | 制約のあるデバイス |
| WebSocket | 双方向リアルタイム | リアルタイム通信 |

### 無線通信規格

| 規格 | 距離 | 特徴 |
|------|------|------|
| Wi-Fi | 中距離 | 高速、高消費電力 |
| Bluetooth/BLE | 短距離 | 低消費電力 |
| Zigbee | 短距離 | メッシュネットワーク |
| LoRaWAN | 長距離 | 低消費電力、低速 |
| LTE-M/NB-IoT | 長距離 | セルラーIoT |

### エッジコンピューティング

クラウドではなくデバイス近くで処理を行う。

| メリット | 説明 |
|---------|------|
| 低遅延 | ローカル処理で応答高速化 |
| 帯域削減 | クラウドへの送信データ削減 |
| プライバシー | データをローカルに保持 |
| オフライン対応 | ネットワーク断でも動作 |

| 特性 | 説明 |
|------|------|

### データウェアハウス

| サービス | プロバイダー |

### データレイク

構造化・非構造化データを生のまま保存。

| 特徴 | 説明 |
|------|------|
| スキーマオンリード | 読み取り時にスキーマ適用 |
| 柔軟性 | あらゆるデータ形式を保存 |
| コスト効率 | 低コストストレージ利用 |

### ETL/ELT

| プロセス | 説明 |
|---------|------|
| Extract | データ抽出 |
| Transform | データ変換 |
| Load | データロード |

- ETL: 変換してからロード（従来型）
- ELT: ロードしてから変換（クラウド向け）

## ブロックチェーン

### 基本概念

| 用語 | 説明 |
|------|------|
| ブロック | トランザクションの集合 |
| チェーン | ブロックのハッシュで連結 |
| 分散台帳 | 複数ノードで同じ台帳を共有 |
| コンセンサス | 合意形成アルゴリズム |

### 特徴

| 特徴 | 説明 |
|------|------|
| 改ざん耐性 | 過去のデータ変更が困難 |
| 透明性 | 全参加者がデータを閲覧可能 |
| 非中央集権 | 単一の管理者が存在しない |
| トレーサビリティ | 履歴の追跡が可能 |

### コンセンサスアルゴリズム

| 方式 | 説明 | 例 |
|------|------|-----|
| PoW | 計算量による証明 | Bitcoin |
| PoS | 保有量による証明 | Ethereum 2.0 |
| PBFT | 投票による合意 | Hyperledger |

### ブロックチェーンの種類

| 種類 | 参加 | 例 |
|------|------|-----|
| パブリック | 誰でも参加可能 | Bitcoin, Ethereum |
| プライベート | 単一組織 | 企業内システム |
| コンソーシアム | 複数組織 | 業界団体 |

### スマートコントラクト

ブロックチェーン上で自動実行されるプログラム。

| プラットフォーム | 言語 |
|----------------|------|
| Ethereum | Solidity |
| Hyperledger | Go, JavaScript |

### 活用分野

| 分野 | 用途 |
|------|------|
| 金融 | 送金、決済 |
| サプライチェーン | トレーサビリティ |
| 不動産 | 権利管理 |
| 医療 | 電子カルテ共有 |
| 行政 | 電子投票 |
