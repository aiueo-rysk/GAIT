# GAIT試験 用語集

GAIT試験で頻出する重要用語をカテゴリ別にまとめています。

---

## インフラストラクチャ

| 用語 | 説明 |
|------|------|
| **RAID** | 複数のディスクを組み合わせて冗長性や性能を向上させる技術。RAID0(ストライピング)、RAID1(ミラーリング)、RAID5(パリティ分散)、RAID6(2重パリティ)など |
| **VLAN** | 物理的な接続に関係なくネットワークを論理的に分割する技術 |
| **NAT** | プライベートIPアドレスとグローバルIPアドレスを変換する技術 |
| **DHCP** | IPアドレスなどのネットワーク設定を自動的に割り当てるプロトコル |
| **DNS** | ドメイン名とIPアドレスを相互変換するシステム。ポート53を使用 |
| **ARP** | IPアドレスからMACアドレスを取得するプロトコル |
| **TCP** | コネクション型で信頼性の高い通信プロトコル。再送制御・順序制御あり |
| **UDP** | コネクションレス型で高速な通信プロトコル。信頼性は低い |
| **NAS** | ネットワーク経由でファイル単位のアクセスを提供するストレージ |
| **SAN** | ブロック単位でアクセスする高速なストレージネットワーク |
| **UPS** | 停電時にバッテリーから電力を供給する無停電電源装置 |
| **ロードバランサー** | 複数サーバーにトラフィックを分散させる装置 |
| **MTBF** | 平均故障間隔。システムの信頼性を表す指標 |
| **MTTR** | 平均修復時間。障害からの復旧にかかる時間 |
| **稼働率** | MTBF / (MTBF + MTTR) で計算される |

## OS・ミドルウェア

| 用語 | 説明 |
|------|------|
| **chmod** | ファイルのアクセス権限を変更するLinuxコマンド |
| **chown** | ファイルの所有者を変更するLinuxコマンド |
| **cron** | 指定した日時にコマンドを自動実行するスケジューラ |
| **systemd** | Linuxのシステム・サービス管理デーモン |
| **プロセス** | 独立したメモリ空間を持つ実行単位 |
| **スレッド** | 同一プロセス内でメモリを共有する軽量な実行単位 |
| **仮想メモリ** | ディスク領域を使用してメモリを拡張する仕組み |
| **LVM** | 論理ボリュームを柔軟に管理できるディスク管理技術 |
| **NTFS** | 暗号化・圧縮・アクセス権制御に対応したWindowsファイルシステム |
| **Active Directory** | Windowsのディレクトリサービス。認証・グループポリシーを提供 |
| **リバースプロキシ** | サーバー側でクライアントからのリクエストを受け、背後のサーバーに転送する仕組み |
| **RabbitMQ** | メッセージキューイングの代表的なミドルウェア |
| **SIGKILL** | プロセスを強制終了するシグナル（シグナル9） |

## データベース

| 用語 | 説明 |
|------|------|
| **ACID特性** | 原子性(A)、一貫性(C)、独立性(I)、永続性(D)。トランザクションの特性 |
| **主キー** | テーブル内で各行を一意に識別。重複・NULLは不可 |
| **外部キー** | 他テーブルの主キーを参照し、参照整合性を維持 |
| **正規化** | データの冗長性を排除する設計手法。第1〜第3正規形など |
| **インデックス** | 検索(SELECT)を高速化するためのデータ構造 |
| **ビュー** | 実テーブルに基づく仮想的なテーブル |
| **ストアドプロシージャ** | データベースサーバー側で実行されるプログラム |
| **デッドロック** | 複数トランザクションが互いにロック解放を待つ状態 |
| **楽観的ロック** | ロック取得せず、更新時にバージョンで競合を検出 |
| **シャーディング** | データを複数DBに水平分割してスケーラビリティを向上 |
| **CAP定理** | 一貫性(C)、可用性(A)、分断耐性(P)のうち2つまで同時に満たせる |
| **MongoDB** | ドキュメント指向のNoSQLデータベース |
| **Redis** | インメモリ型のキー・バリューストア |

## セキュリティ

| 用語 | 説明 |
|------|------|
| **CIA** | 機密性(C)、完全性(I)、可用性(A)。情報セキュリティの三要素 |
| **公開鍵暗号** | 公開鍵で暗号化、秘密鍵で復号。鍵配送問題を解決 |
| **共通鍵暗号** | 同じ鍵で暗号化・復号。AESが代表例 |
| **ハッシュ関数** | 固定長のハッシュ値を生成。SHA-256など |
| **SQLインジェクション** | 悪意のあるSQLを注入する攻撃。対策はプリペアドステートメント |
| **XSS** | 悪意のあるスクリプトを埋め込む攻撃。対策はエスケープ処理 |
| **CSRF** | ユーザーの意図しないリクエストを送信させる攻撃 |
| **DDoS** | 多数のコンピュータからサービスを妨害する攻撃 |
| **ランサムウェア** | データを暗号化して身代金を要求するマルウェア |
| **ゼロトラスト** | すべてのアクセスを検証するセキュリティモデル |
| **多要素認証** | 異なる種類の要素（知識・所持・生体）を組み合わせた認証 |
| **SSO** | 一度の認証で複数システムにアクセス可能にする仕組み |
| **WAF** | Webアプリケーション層の攻撃を防ぐファイアウォール |
| **IDS/IPS** | 侵入検知(IDS)・侵入防止(IPS)システム |
| **SIEM** | セキュリティログの収集・分析・相関付けを行うシステム |
| **PKI** | 公開鍵基盤。認証局(CA)がデジタル証明書を発行 |

## 計算量（オーダー記法）
