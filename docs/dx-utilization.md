## DXの基本

### DXの定義

経済産業省の定義：
> 企業がビジネス環境の激しい変化に対応し、データとデジタル技術を活用して、顧客や社会のニーズを基に、製品やサービス、ビジネスモデルを変革するとともに、業務そのものや、組織、プロセス、企業文化・風土を変革し、競争上の優位性を確立すること

### デジタル化の段階

| 段階 | 説明 | 例 |
|------|------|-----|
| デジタイゼーション | アナログ→デジタル変換 | 紙→PDF化 |
| デジタライゼーション | プロセスのデジタル化 | ワークフロー電子化 |
| DX | ビジネスモデルの変革 | 新サービス創出 |

### DXの阻害要因

| 要因 | 説明 |
|------|------|
| レガシーシステム | 老朽化したシステムの維持負担 |
| IT人材不足 | DX推進人材の確保困難 |
| 経営層の理解不足 | 投資判断の遅れ |
| 組織の縦割り | 部門間連携の困難さ |
| データ活用の遅れ | データの分散・サイロ化 |

## プロジェクトマネジメント

### プロジェクトの制約（トリプルコンストレイント）

```
      スコープ
        ／＼
       ／  ＼
      ／    ＼
   コスト ── 時間
```

品質はこの3つのバランスで決まる。

### PMBOK

プロジェクトマネジメント知識体系。

**プロセス群**
| プロセス群 | 内容 |
|-----------|------|
| 立上げ | プロジェクト認可 |
| 計画 | 計画策定 |
| 実行 | 計画の実行 |
| 監視・コントロール | 進捗管理・是正 |
| 終結 | 完了・クローズ |

**知識エリア**
| エリア | 内容 |
|--------|------|
| 統合 | プロジェクト全体の調整 |
| スコープ | 作業範囲の管理 |
| スケジュール | 日程管理 |
| コスト | 予算管理 |
| 品質 | 品質管理 |
| 資源 | 人員・設備管理 |
| コミュニケーション | 情報共有 |
| リスク | リスク管理 |
| 調達 | 外部調達管理 |
| ステークホルダー | 関係者管理 |

### 見積り手法

| 手法 | 説明 |
|------|------|
| 類推見積り | 過去の類似案件から推定 |
| ボトムアップ見積り | 作業を分解して積み上げ |
| パラメトリック見積り | 統計モデルで算出 |
| 3点見積り | 最善・最悪・最頻値の加重平均 |

### WBS（Work Breakdown Structure）

作業を階層的に分解した構造。

```
プロジェクト
├── フェーズ1
│   ├── タスク1.1
│   │   ├── サブタスク1.1.1
│   │   └── サブタスク1.1.2
│   └── タスク1.2
└── フェーズ2
    └── タスク2.1
```

### スケジュール管理

| 手法 | 説明 |
|------|------|
| ガントチャート | 横棒グラフでスケジュール表示 |
| PERT | 作業の依存関係をネットワーク図で表現 |
| クリティカルパス | 最長経路（遅延不可の作業列） |
| マイルストーン | 重要な節目 |
