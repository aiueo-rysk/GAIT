## プログラミング基礎

### プログラミングパラダイム

| パラダイム | 特徴 | 代表言語 |
|-----------|------|---------|
| 手続き型 | 処理を順番に記述 | C, Pascal |
| オブジェクト指向 | データと処理をまとめる | Java, C++, Python |
| 関数型 | 関数の組み合わせ | Haskell, Scala |
| 論理型 | 論理式で記述 | Prolog |

### オブジェクト指向の三大要素

| 要素 | 説明 |
|------|------|
| カプセル化 | データと処理をまとめ、外部から隠蔽 |
| 継承 | 既存クラスの機能を引き継ぐ |
| ポリモーフィズム | 同じインターフェースで異なる動作 |

### 主要プログラミング言語

| 言語 | 特徴 | 用途 |
|------|------|------|
| Java | クロスプラットフォーム、静的型付け | 業務システム、Android |
| Python | シンプル、動的型付け | AI/ML、Web、スクリプト |
| JavaScript | ブラウザで動作、非同期処理 | Web（フロント/バック） |
| C# | Microsoft製、.NET Framework | Windows、ゲーム（Unity） |
| Go | シンプル、高速、並行処理 | インフラ、マイクロサービス |
| Rust | メモリ安全、高速 | システムプログラミング |

## データ構造とアルゴリズム

### 基本データ構造

| データ構造 | 特徴 | 計算量（検索） |
|-----------|------|---------------|
| 配列 | 連続したメモリ領域 | O(n) |
| 連結リスト | ポインタで連結 | O(n) |
| スタック | LIFO（後入れ先出し） | O(n) |
| キュー | FIFO（先入れ先出し） | O(n) |
| ハッシュテーブル | キーから直接アクセス | O(1) |
| 木構造 | 階層構造 | O(log n) |

### 計算量（オーダー記法）

| 記法 | 名称 | 例 |
|------|------|-----|
| O(1) | 定数時間 | 配列のインデックスアクセス |
| O(log n) | 対数時間 | 二分探索 |
| O(n) | 線形時間 | 線形探索 |
| O(n log n) | 準線形時間 | マージソート |
| O(n²) | 二乗時間 | バブルソート |

### ソートアルゴリズム

| アルゴリズム | 平均計算量 | 特徴 |
|-------------|-----------|------|
| バブルソート | O(n²) | 単純だが遅い |
| クイックソート | O(n log n) | 高速、不安定 |
| マージソート | O(n log n) | 安定、追加メモリ必要 |
| ヒープソート | O(n log n) | 安定しない |

## 開発手法

### ウォーターフォール型

要件定義 → 設計 → 実装 → テスト → 運用

| 特徴 | 説明 |
|------|------|
| メリット | 計画が立てやすい、ドキュメント充実 |
| デメリット | 変更に弱い、後戻りコストが高い |
| 適用場面 | 要件が明確な大規模プロジェクト |

### アジャイル開発

短いイテレーションで開発・リリースを繰り返す。

| 手法 | 特徴 |
|------|------|
| スクラム | スプリント単位、デイリースクラム |
| XP | ペアプログラミング、TDD |
| カンバン | 可視化、WIP制限 |

**スクラムの役割**
| 役割 | 責務 |
|------|------|
| プロダクトオーナー | プロダクトバックログ管理 |
| スクラムマスター | プロセス改善、障害除去 |
| 開発チーム | 自己組織化、成果物作成 |

### DevOps

開発（Development）と運用（Operations）の統合。

| プラクティス | 説明 |
|-------------|------|
| CI/CD | 継続的インテグレーション/デリバリー |
| Infrastructure as Code | インフラをコードで管理 |
| モニタリング | 継続的な監視・改善 |

## テスト

### テストレベル

| レベル | 対象 | 実施者 |
|--------|------|--------|
| 単体テスト | 関数・メソッド | 開発者 |
| 結合テスト | モジュール間連携 | 開発者 |
| システムテスト | システム全体 | QA |
| 受入テスト | ビジネス要件 | 顧客 |

### テスト技法

| 技法 | 説明 |
|------|------|
| ホワイトボックス | 内部構造を考慮 |
| ブラックボックス | 入出力のみ確認 |
| 境界値分析 | 境界値を重点的にテスト |
| 同値分割 | 同じ結果になる入力をグループ化 |

### テスト自動化

| ツール | 用途 |
|--------|------|
| JUnit | Java単体テスト |
| pytest | Python単体テスト |
| Selenium | ブラウザ自動テスト |
| Jest | JavaScript単体テスト |
