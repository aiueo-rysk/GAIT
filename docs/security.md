## 情報セキュリティの三要素（CIA）

| 要素 | 説明 |
|------|------|
| 機密性（Confidentiality） | 許可された人だけがアクセス可能 |
| 完全性（Integrity） | 情報が正確で改ざんされていない |
| 可用性（Availability） | 必要な時にアクセス可能 |

### 追加の要素

| 要素 | 説明 |
|------|------|
| 真正性（Authenticity） | 本人であることの確認 |
| 責任追跡性（Accountability） | 行動を追跡可能 |
| 否認防止（Non-repudiation） | 行為を否定できない |
| 信頼性（Reliability） | 意図した動作の保証 |

## 認証と認可

### 認証の3要素

| 要素 | 説明 | 例 |
|------|------|-----|
| 知識情報 | 知っていること | パスワード、PIN |
| 所持情報 | 持っていること | ICカード、スマホ |
| 生体情報 | 本人の特徴 | 指紋、顔認証 |

### 認証方式

| 方式 | 説明 |
|------|------|
| 多要素認証（MFA） | 2つ以上の要素で認証 |
| シングルサインオン（SSO） | 1回の認証で複数サービス利用 |
| SAML | XML based認証プロトコル |
| OAuth 2.0 | 認可フレームワーク |
| OpenID Connect | OAuth 2.0ベースの認証 |

### 認証 vs 認可

| 概念 | 説明 |
|------|------|
| 認証（Authentication） | 「誰であるか」の確認 |
| 認可（Authorization） | 「何ができるか」の許可 |

## 暗号化

### 暗号化の種類

| 種類 | 特徴 | 用途 |
|------|------|------|
| 共通鍵暗号 | 暗号化・復号に同じ鍵 | 大量データ暗号化 |
| 公開鍵暗号 | 公開鍵と秘密鍵のペア | 鍵交換、デジタル署名 |
| ハイブリッド暗号 | 両者を組み合わせ | TLS/SSL |

### 主要アルゴリズム

**共通鍵暗号**
| アルゴリズム | 特徴 |
|-------------|------|
| AES | 現在の標準（128/192/256bit） |
| DES | 古い規格（56bit、脆弱） |
| 3DES | DESを3回適用 |

**公開鍵暗号**
| アルゴリズム | 特徴 |
|-------------|------|
| RSA | 素因数分解の困難性 |
| 楕円曲線暗号（ECC） | 短い鍵で高セキュリティ |

### ハッシュ関数

| アルゴリズム | 出力長 | 状態 |
|-------------|--------|------|
| MD5 | 128bit | 非推奨（衝突発見） |
| SHA-1 | 160bit | 非推奨 |
| SHA-256 | 256bit | 推奨 |
| SHA-3 | 可変 | 最新 |

### デジタル署名

1. 送信者が秘密鍵でハッシュ値を暗号化（署名）
2. 受信者が公開鍵で復号して検証

**効果**
- 改ざん検知
- 送信者の認証
- 否認防止

### PKI（公開鍵基盤）

| 要素 | 役割 |
|------|------|
| CA（認証局） | 証明書の発行 |
| RA（登録局） | 申請の審査 |
| 証明書 | 公開鍵と所有者の紐付け |
| CRL | 失効証明書リスト |

## サイバー攻撃

### 攻撃の分類

| カテゴリ | 攻撃例 |
|---------|--------|
| ネットワーク攻撃 | DoS/DDoS, 中間者攻撃 |
| Web攻撃 | SQLインジェクション, XSS |
| マルウェア | ウイルス, ランサムウェア |
| ソーシャルエンジニアリング | フィッシング, 標的型攻撃 |

### 主要な攻撃手法

**DoS/DDoS攻撃**
| 種類 | 説明 |
|------|------|
| SYN Flood | TCP接続要求を大量送信 |
| UDP Flood | UDPパケットを大量送信 |
| Amplification | 増幅効果を利用（DNS等） |

**Webアプリケーション攻撃**
| 攻撃 | 説明 | 対策 |
|------|------|------|
| SQLインジェクション | SQL文を挿入 | プレースホルダ使用 |
| XSS | 悪意のあるスクリプト挿入 | エスケープ処理 |
| CSRF | 偽のリクエストを送信 | トークン検証 |
| ディレクトリトラバーサル | パス操作でファイルアクセス | 入力検証 |

**マルウェア**
| 種類 | 特徴 |
|------|------|
| ウイルス | 他のファイルに寄生 |
| ワーム | 自己複製してネットワーク拡散 |
| トロイの木馬 | 正規ソフトに偽装 |
| ランサムウェア | データを暗号化して身代金要求 |
| スパイウェア | 情報を収集・送信 |
| ボット | 遠隔操作される |

**ソーシャルエンジニアリング**
| 手法 | 説明 |
|------|------|
| フィッシング | 偽サイトに誘導 |
| スピアフィッシング | 特定対象を狙った攻撃 |
| ビジネスメール詐欺（BEC） | 経営者等を装う |
| プリテキスティング | 身分を偽って情報収集 |

## セキュリティ対策

### ネットワークセキュリティ

| 対策 | 説明 |
|------|------|
| ファイアウォール | 通信の許可・遮断 |
| IDS/IPS | 侵入検知/防止 |
| WAF | Webアプリケーションファイアウォール |
| VPN | 暗号化された仮想ネットワーク |
| DMZ | 非武装地帯（外部公開サーバー配置） |

### エンドポイントセキュリティ

| 対策 | 説明 |
|------|------|
| アンチウイルス | マルウェア検知・駆除 |
| EDR | Endpoint Detection and Response |
| DLP | Data Loss Prevention（情報漏洩防止） |
| ディスク暗号化 | ストレージの暗号化 |

### セキュアコーディング

| 原則 | 説明 |
|------|------|
| 入力検証 | すべての入力を検証 |
| 出力エスケープ | 出力時に適切にエスケープ |
| 最小権限 | 必要最低限の権限のみ付与 |
| 多層防御 | 複数の防御層を設置 |
| エラー処理 | 適切なエラーハンドリング |
